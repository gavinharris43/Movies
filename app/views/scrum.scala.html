What is Scrum

Scrum is a framework within which people can address complex adaptive problems, while productively and creatively delivering products of the highest possible value. Scrum itself is a simple framework for effective team collaboration on complex products. Scrum co-creators Ken Schwaber and Jeff Sutherland have written The Scrum Guide to explain Scrum clearly and succinctly. This Guide contains the definition of Scrum. How we used Scrum

With our project we incorporated Scrum into our daily lives

Sprint Planning

this is where we would point tickets that where to be completed within a sprint

Stand Up

During the sprint we would have a Daily Scrum where we would discuss;
What we did the previous day?
What we would be planning to do today?
Any blockers?

Sprint Review

This is where we would review the work completed within the sprint and create any new tickets/ move tickets to the next sprint that were not completed.

Sprint Retrospective

For this part we discussed as a team;
What went well?
What didn't go so well?
What could we do better?
With the suggested improvements we would make actions that should be completed and reviewed in the following sprint retrospective.

Our Results

With use of scrum in this way we where able to have good communication, and it was easier to find when people had issues, therefor making it easier to solve those issues.
With the extra planning, this opened the opportunity for more developers to have influence on the ticket, which in turn helps to clarify what needs done on a ticket.
With using sprints, this has helped our time management as being able to point tickets, to show if we would expect it to be done in a sprint.
